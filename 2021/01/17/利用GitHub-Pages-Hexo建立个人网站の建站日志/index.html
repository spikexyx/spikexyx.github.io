GitHub上创建个人仓库

登录到 GitHub，(如果没有GitHub账号的话，使用邮箱注册一个即可)，点击 new repository 创建新仓库，注意仓库名应为： 用户名.github.io ，(此处用户名即为你的GitHub账号用户名，比如我的用户名为spikexyx，那么这个仓库名就应为spikexyx.github.io)。具体建立仓库的流程等如果存在疑惑请查阅 GitHub 相关文档。

本地安装Git

从Git官网下载：Git - Downloading Package，选择适合自己机型的安装包下载（现在的Windows机子都是64位的，如果不清楚自己的机型就按64位下载），下载后安装即可。安装完成后，需要将你的 Git 和 GitHub 账号绑定，鼠标右键点击，打开 Git Bash

备注： 这里之所以设置 GitHub 密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在 GitHub 上，私钥放置在自己的电脑里。GitHub 要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了ssh，当你推送的时候，git 就会匹配你的私钥跟 GitHub 上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。

获得个人网站域名并绑定（此步可省）

我是从阿里云中注册的域名：阿里云-为了无法计算的价值 申请入口：域名注册，购买域名是建站中唯一一次花钱。当然可以选择比较便宜的域名，比如 .top 等，或者也可以通过其他途径白嫖免费域名，比如 .cf，.me等，具体方法可以去知乎或者谷歌等搜索。

以下仅讲解阿里云中购买域名后的操作：

1.域名解析

购买域名之后，进入阿里云域名控制台，找到你购买的域名，点击解析

然后添加两条解析记录

记录类型都是 CNAME，记录值是你的仓库名称（比如我的是 spikexyx.github.io），主机记录分别是 www 和 @ ，如下图所示：

添加或修改解析记录，都需要等 10 分钟才能生效。

2.获取SSL证书（此步可省）

进入阿里云SSL证书界面，从左侧栏中进入证书资源包，获取免费SSL证书，具体操作参见网站指南。（土豪也可以选择直接购买高级SSL证书，安全性更强）

3.进入GitHub仓库绑定个人域名

登录 GitHub，进入之前创建的仓库，点击 settings，在 Custom domain 中输入你的个人域名：(注意将示例图中的 ivyli.top 换成你自己买的域名)

同时，如果域名已经获取了SSL证书，就启用 Enforce HTTPS，使网站只能通过 https 访问。

4.本地设置CNAME文件

进入本地博客文件夹的 source 目录下，创建一个记事本文件，输入你的个人域名，保存为 CNAME，注意保存成所有文件而不是 txt 文件。

备注：如果没办法直接创建“所有文件”类型，可以去 github 仓库下载 CNAME 文件，然后复制粘贴到 source 目录中。

Tips

  1. GitHub从国内可能访问较慢，通过 fan qiang 或者连上学校VPN可以加快速度。

  2. 注意文章中的 _config.yml 文件在根目录和themes/next目录中分别有一个，两者分别称为站点配置文件和主题配置文件，别搞混了。

  3. 购买域名后要实名认证，否则网站无效。

  4. 购买域名后需要一段时间审核，审核期间网站是挂起的（也就是无效的），审核期间无法添加SSL证书，也无法通过该域名访问博客网站。推荐首先购买域名等待审核期间进行其余工作。

  5. 文章中更换主题以 NexT 主题为例，也可以去 hexo 网站选择其他主题，进入对应主题界面看怎么下载部署。
